Font ligature rendering test
============================

Use this file with a programming font that supports ligatures, such as
Cascadia Code, Fira Code, JetBrains Mono, Hasklig, Iosevka, or Monaspace.

Each line keeps the raw character sequence visible in the left label, then
prints the sequence repeatedly so shaping, spacing, and cursor width issues are
easy to spot.


Arrows
------

fat right arrow    =>    => => =>    a=>b    fn x => x + 1
long fat arrow     ==>   ==> ==>     a==>b   input ==> output
left arrow         <-    <- <- <-    a<-b    end <- middle <- start
long left arrow    <--   <-- <--     a<--b   end <-- middle <-- start
fat left arrow     <=    <= <= <=    a<=b    lower <= value <= upper
long fat left      <==   <== <==     a<==b   output <== input
both directions    <->   <-> <->     a<->b   read <-> write
double compare     <=>   <=> <=>     a<=>b   left <=> right

Cursor width probes
-------------------
The brackets should stay aligned when the terminal advances by the original
character count rather than the rendered glyph width.
